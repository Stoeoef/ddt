Detailed Description

Offers access to all parts of the application's model.

The MainModel models the application's state. It stores the current speakers, debates and solution. Furthermore, it allows various operations on the model elements.

Q_INVOKABLE QStringList MainModel::errors ( )
inline

Returns a list of error descriptions.

The errors need to be fixed before a solution can be generated. They can be presented to the users and give clues why no solution exists (see solutionState). The return value is undefined if the solution state is different than NO_SOLUTION_EXISTS .

void MainModel::generateNewSolution ( )

Generates a new solution and stores it internally.

Call initSolver(); before attempting to call this method. The solution state will be set to CALCULATING, once the solution is finished the state changes to NO_SOLUTION_EXISTS or SOLUTION_EXISTS. The generated solution is stored internally and can be read with getSolution.
